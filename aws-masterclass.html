Module 01 · Foundation

What is Cloud Computing?

Cloud என்றால் என்ன?
⬤ Basic
English

Cloud Computing means renting computers, storage, and software over the internet instead of owning physical hardware. You pay only for what you use, just like electricity — you don't own the power plant, but you get power whenever you need it.

தமிழ்

Cloud Computing என்பது உங்கள் சொந்த கணினி வாங்காமல் இணையத்தில் வாடகைக்கு கணினி, சேமிப்பு, மென்பொருள் பயன்படுத்துவது. மின்சாரம் போல் — நீங்கள் Power Plant வைத்திருக்க மாட்டீர்கள், ஆனால் தேவைப்படும்போது மின்சாரம் கிடைக்கும். அதே போல் Cloud-ல் தேவையான போது Server கிடைக்கும்.

Swiggy / Zomato Example: When Swiggy started, they didn't buy 10,000 servers. Instead they rented servers from AWS. During Diwali when orders spike 10x, they scale up instantly. After festival, scale back down. They only paid for what they used. If they had bought physical servers, 90% would sit idle most of the year — wasted money.

☁ Cloud vs Traditional Infrastructure

Types of Cloud Services 3 Models

IaaS — Infrastructure as a Service
Rent raw servers, networking, storage. You manage OS and apps. Like renting a bare apartment — you bring your furniture. Example: EC2.
PaaS — Platform as a Service
Rent a managed platform to deploy your code. Like renting a furnished apartment. Example: AWS Elastic Beanstalk, Heroku.
SaaS — Software as a Service
Use ready-made software. Like staying in a hotel. Example: Gmail, Zoom, Salesforce.

AWS என்றால் என்ன?
⬤ Basic
English

Amazon Web Services (AWS) is the world's largest and most comprehensive cloud platform, owned by Amazon. It offers 200+ services including computing, storage, databases, AI, analytics, security, and more. Companies of all sizes — from startups to Netflix and NASA — use AWS to power their applications.

Amazon Web Services (AWS) என்பது Amazon நிறுவனம் நடத்தும் உலகின் மிகப்பெரிய cloud platform. இதில் 200-க்கும் மேல் சேவைகள் உள்ளன — கணினி, சேமிப்பு, Database, AI, Security எல்லாமே. Netflix, NASA, Swiggy, Flipkart போன்ற நிறுவனங்கள் AWS-ஐ பயன்படுத்துகின்றன.

Why Netflix uses AWS: Netflix streams 3.3 billion hours of video per month. They use AWS to handle traffic in 190 countries. If a new season of a show drops and 50 million people watch simultaneously, AWS auto-scales to handle it. Netflix doesn't own a single data center — it's 100% on AWS.

Module 03 · Foundation

AWS Global Infrastructure

Regions · Availability Zones
⬤ Basic
English

Region = A geographic area with multiple data centers (e.g., ap-south-1 is Mumbai). Availability Zone (AZ) = An isolated data center within a Region. Multiple AZs protect against single points of failure. If one AZ catches fire, others keep your app running.

தமிழ்

Region = ஒரு நாட்டிலோ / பகுதியிலோ AWS Data Center இருக்கும் இடம் (உதாரணம்: Mumbai = ap-south-1). Availability Zone = Region-க்கு உள்ளே தனித்தனியான Data Center buildings. ஒரு building தீயில் எரிந்தாலும் மற்றது உங்கள் app-ஐ இயக்கும்.

☁ AWS Global Infrastructure Hierarchy

Chennai User Example: If you're in Puducherry and open an app deployed in ap-south-1 (Mumbai), your request travels ~1300 km — very fast. But if the app were only in us-east-1 (Virginia, USA), the request would travel 13,000 km — 10x slower. That's why choosing the right Region matters!

Module 04 · Compute

EC2 — Elastic Compute Cloud

Virtual Servers on AWS
⬤ Basic → Intermediate
English

EC2 is AWS's virtual machine service. It lets you rent a computer (called an Instance) in the cloud with any OS, CPU, RAM, and storage you need. You can launch it in seconds, use it for an hour or years, and terminate when done. You're only charged for the time it runs.

தமிழ்

EC2 என்பது AWS-இன் Virtual Machine service. Cloud-ல் ஒரு computer (Instance என்று சொல்வார்கள்) வாடகைக்கு எடுக்கலாம். எந்த OS வேண்டும், எவ்வளவு RAM வேண்டும் — நீங்களே தேர்வு செய்யலாம். Seconds-ல் start ஆகும், தேவை இல்லாவிட்டால் stop செய்யலாம். இயங்கிய நேரத்திற்கு மட்டும் பணம் செலுத்த வேண்டும்.

Real Example — Startup: A startup in Chennai builds a Django app. They launch a t3.medium EC2 instance (2 vCPU, 4GB RAM) for ₹2,500/month. When their app is featured in the news and traffic spikes 50x, they launch 10 more instances in minutes. After the spike, they terminate the extras. Total cost: proportional to actual usage.

Security Groups Firewall

English

A Security Group is a virtual firewall for your EC2 instance. It controls which traffic can come IN (inbound) and go OUT (outbound) based on protocol, port, and IP address. By default, all inbound traffic is DENIED.

தமிழ்

Security Group என்பது உங்கள் EC2-இன் virtual firewall. எந்த traffic உள்ளே வரலாம் (inbound), எது போகலாம் (outbound) என்று rules set செய்வது. Default-ஆக எல்லா inbound traffic-ம் BLOCK ஆகும். நீங்கள் explicitly allow செய்ய வேண்டும்.

Module 05 · Storage

S3 — Simple Storage Service

Buckets · Objects · Static Hosting
⬤ Basic → Intermediate
English

S3 is AWS's object storage service. You store files (called Objects) inside containers called Buckets. Think of a Bucket as a folder in the cloud. Each Object gets a unique URL. S3 is infinitely scalable — store 1 file or 1 trillion files with no configuration changes.

தமிழ்

S3 என்பது AWS-இன் file storage service. Bucket என்பது cloud-ல் ஒரு folder போன்றது. அதற்குள் Objects (files) சேமிக்கலாம். Photo, Video, PDF, JSON எதுவும் store செய்யலாம். ஒரு file-ஓ, ஒரு trillion files-ஓ — எந்த setting மாற்றாமல் scale ஆகும்.

Real Example — OTT Platform: An OTT platform (like Hotstar) stores all their movies and shows in S3. A 4K movie = ~100GB. S3 stores it with 99.999999999% (11 nines) durability. The video player fetches from S3 via CloudFront CDN. Users in Chennai, Delhi, Mumbai all get fast stream from the nearest edge location. Cost: ~$0.023 per GB/month (less than 2 rupees per GB).

S3 Concepts Core

Bucket
Global-unique named container. Like a domain name — must be unique across all AWS accounts worldwide. Example: my-company-assets-2024
Object
Any file stored in S3. Max single file = 5TB. Each object has a key (path), value (data), metadata, and a URL.
Static Website Hosting
Host a React/Vue/HTML app directly from S3. No server needed. Enable in bucket settings. Cost = near zero for small sites.
Storage Classes
Standard (frequent), Infrequent Access, Glacier (archive). Automatically move old files to cheaper storage with Lifecycle Rules.
# Create an S3 bucket aws s3 mb s3://my-app-bucket-pondicherry --region ap-south-1 # Upload a file aws s3 cp index.html s3://my-app-bucket-pondicherry/ # Sync a folder (e.g., React build) aws s3 sync ./build s3://my-app-bucket-pondicherry/ --delete # Enable static website hosting aws s3 website s3://my-app-bucket-pondicherry/ \ --index-document index.html \ --error-document 404.html # URL: http://my-app-bucket-pondicherry.s3-website.ap-south-1.amazonaws.com

Module 06 · Security

IAM — Identity & Access Management

Users · Groups · Roles · Policies
⬤ Intermediate
English

IAM controls who can access what in your AWS account. Users are people (with username/password + access keys). Groups are collections of users sharing the same permissions. Roles are temporary identities assumed by services or humans. Policies are JSON documents that define what actions are allowed or denied.

தமிழ்

IAM என்பது உங்கள் AWS account-ல் யாருக்கு என்ன permission என்று control செய்வது. Users = மனிதர்கள் (developer, devops). Groups = பல users-ஐ ஒரு team-ஆக வைத்து same permission கொடுப்பது. Roles = EC2 போன்ற services-க்கு temporary permission கொடுப்பது. Policies = JSON-ல் எழுதப்பட்ட permission rules.

Office Building Analogy / அலுவலக உதாரணம்:
• Users = Employees (ஊழியர்கள்) with ID cards
• Groups = "DevOps Team" — all DevOps employees get server room access
• Roles = A contractor badge valid only today (temporary access)
• Policies = The rule book: "Finance team can enter Floor 3 only, 9am-6pm"

🔐 IAM Architecture

Principle of Least Privilege Best Practice

Always grant the minimum permissions needed. A developer who only needs to read S3 should NOT have admin rights. This limits blast radius if an account is compromised.

# Example: Read-only S3 Policy { "Version": "2012-10-17", "Statement": [{ "Effect": "Allow", "Action": [ "s3:GetObject", "s3:ListBucket" // NO s3:DeleteObject, s3:PutObject! ], "Resource": "arn:aws:s3:::my-app-bucket/*" }] }

Module 07 · Networking

VPC — Virtual Private Cloud

Subnets · Internet Gateway · Private Network
⬤ Intermediate
English

A VPC is your own isolated network inside AWS. Like a private gated apartment complex with its own roads, security, and rooms. Within a VPC, you create Subnets — smaller network segments. Public subnets have internet access. Private subnets are completely isolated from the internet — perfect for databases.

தமிழ்

VPC என்பது AWS-ல் உங்களுக்கு மட்டும் சொந்தமான private network. ஒரு gated apartment complex போல் — உள்ளே தனி roads, security, rooms உள்ளது. VPC-க்கு உள்ளே Subnets உருவாக்கலாம். Public Subnet = internet-உடன் இணைக்கப்பட்டது (web servers). Private Subnet = internet இல்லாமல் isolated (databases, backend).

🕸️ VPC Architecture — 3-Tier Web Application

Bank Security Model: Your app's database (RDS) lives in a Private Subnet — no internet access AT ALL. Hackers can't reach it directly. The web server in the Public Subnet talks to the DB through internal network. This is exactly how banks secure their core banking systems. The ATM (public) talks to the backend (private), but internet users can't reach the backend directly.

Module 08 · Database

RDS — Relational Database Service

PostgreSQL · MySQL · Backups · Multi-AZ
⬤ Intermediate
English

RDS is a fully managed relational database service. AWS handles installation, patching, backups, and replication. You focus on your data, not the DBA work. Supports PostgreSQL, MySQL, MariaDB, Oracle, SQL Server. Multi-AZ creates a synchronous standby replica in another AZ for automatic failover within ~60 seconds.

தமிழ்

RDS என்பது AWS நிர்வகிக்கும் database service. நீங்கள் database install செய்ய வேண்டாம், backup எடுக்க வேண்டாம் — AWS எல்லாம் செய்யும். PostgreSQL, MySQL போன்ற databases support செய்கிறது. Multi-AZ = ஒரு AZ-ல் database fail ஆனால் மற்றொரு AZ-ல் உள்ள copy 60 seconds-ல் active ஆகும்.

Multi-AZ vs Read Replica High Availability

FeatureMulti-AZRead Replica
PurposeHigh Availability (HA) — zero downtime failoverRead performance scaling
Sync TypeSynchronous — data matches instantlyAsynchronous — slight lag
Can be read from?No — standby onlyYes — route read queries here
FailoverAutomatic in ~60 secondsManual promotion
தமிழ்Crash ஆனாலும் உடனே backup database active ஆகும்Read queries-ஐ பகிர்ந்துகொள்ள

Hospital System Example: A hospital's patient records database uses Multi-AZ RDS. If the primary database in AZ-1a has a hardware failure at 2am, AWS automatically promotes the standby in AZ-1b within 60 seconds. Doctors can still access patient records. Without Multi-AZ, the hospital system would be down until an engineer fixes the hardware — potentially hours. For critical systems, Multi-AZ is non-negotiable.

# Create RDS PostgreSQL with Multi-AZ aws rds create-db-instance \ --db-instance-identifier prod-postgres \ --db-instance-class db.t3.medium \ --engine postgres \ --engine-version 15.4 \ --master-username admin \ --master-user-password "SecurePass123!" \ --allocated-storage 100 \ --multi-az \ # Enable Multi-AZ failover --backup-retention-period 7 \ # Keep backups for 7 days --storage-encrypted \ # Encrypt data at rest --vpc-security-group-ids sg-db123 \ --db-subnet-group-name private-subnet-group

Module 09 · Scaling

Load Balancer & Auto Scaling

High Availability · Traffic Distribution
⬤ Intermediate
English

Load Balancer distributes incoming traffic across multiple EC2 instances, so no single server gets overwhelmed. Auto Scaling Group (ASG) automatically adds EC2 instances when traffic is high and removes them when traffic drops — you only pay for what you need, and you never go down due to traffic spikes.

தமிழ்

Load Balancer என்பது user requests-ஐ பல EC2 servers-க்கு சமமாக பகிர்வது. ஒரு server மட்டும் overload ஆகாமல் பார்க்கும். Auto Scaling என்பது traffic அதிகரித்தால் தானே புதிய EC2 instances start ஆகும், traffic குறைந்தால் terminate ஆகும். IPL match time-ல் கூடும், பிறகு குறையும்.

⚖️ Load Balancer + Auto Scaling Flow

Big Billion Day / Flipkart Sale: Flipkart's servers handle 50,000 orders/minute on normal days. On Big Billion Day, it spikes to 5,00,000/minute — 10x more. With Auto Scaling, Flipkart's ASG adds 90 more EC2 instances automatically when CPU usage crosses 70%. The Load Balancer spreads traffic evenly. After the sale ends, the extra instances are terminated. Flipkart only pays for the extra servers for those 12 hours.

Module 10 · DNS

Route 53 — DNS Service

Domain Names · Routing · Health Checks
⬤ Intermediate
English

Route 53 is AWS's highly available DNS (Domain Name System) service. DNS translates human-readable domain names (myapp.com) into IP addresses (13.234.56.78) that computers understand. Route 53 also supports advanced routing — latency-based (send user to nearest server), failover (if primary is down, switch to backup), and weighted routing.

தமிழ்

Route 53 என்பது AWS-இன் DNS service. DNS என்பது phone book போன்றது — myapp.com என்று type செய்தால் அது 13.234.56.78 என்ற IP-க்கு திருப்பி விடும். Advanced routing: User Chennai-ல் இருந்தால் Mumbai server-க்கு, US-ல் இருந்தால் Virginia server-க்கு automatically route செய்யும்.

Simple Routing
myapp.com → one IP. Basic mapping. Good for single server setup.
Latency-Based
Send user to the AWS region with lowest latency. Indian users → Mumbai, US users → Virginia. Automatic.
Failover Routing
Primary server down? Route 53 health checks detect it and switch DNS to the backup server within 60 seconds.
Weighted Routing
Send 10% of traffic to new version, 90% to old. Perfect for canary deployments / A-B testing new features safely.

Module 11 · Containers

ECS — Elastic Container Service

Docker on AWS · Tasks · Services · Clusters
⬤ Advanced
English

ECS is AWS's managed container orchestration service for Docker. A Task is a running Docker container (one instance of your app). A Service ensures X number of tasks are always running, replaces failed ones, and integrates with the Load Balancer. A Cluster is the logical grouping of Tasks and Services.

தமிழ்

ECS என்பது Docker containers-ஐ AWS-ல் நிர்வகிக்கும் service. Task = ஒரு இயங்கும் Docker container. Service = எப்போதும் 3 tasks இயங்க வேண்டும் என்று சொல்லி வைப்பது — ஒன்று crash ஆனால் புதுது start ஆகும். Cluster = இந்த எல்லா tasks/services-ஐயும் organize செய்யும் group.

Module 12 · Kubernetes

EKS — Elastic Kubernetes Service

Pods · Deployments · Services
⬤ Advanced
English

EKS is AWS's managed Kubernetes service. Kubernetes (K8s) is the industry-standard container orchestration platform. A Pod is the smallest unit — one or more containers running together. A Deployment manages pods declaratively (desired state). A Service exposes pods with a stable network endpoint.

தமிழ்

EKS என்பது AWS-இல் Kubernetes நிர்வகிக்கும் service. Kubernetes என்பது containers-ஐ large scale-ல் manage செய்யும் tool. Pod = ஒரு அல்லது சில containers இணைந்து இயங்கும் smallest unit. Deployment = "3 pods எப்போதும் இயங்க வேண்டும்" என்று declare செய்வது. Service = Pods-ஐ network மூலம் access செய்ய ஒரு stable endpoint.

Module 13 · Observability

CloudWatch

Logs · Metrics · Monitoring · Alerts · Dashboards
⬤ Advanced
English

CloudWatch is AWS's monitoring and observability service. It collects Logs (text output from your app), Metrics (numbers like CPU%, request count, latency), creates Alarms that trigger notifications or actions when a metric crosses a threshold, and shows everything in visual Dashboards.

தமிழ்

CloudWatch என்பது AWS-இன் monitoring service. இது Logs (app-இன் text output), Metrics (CPU%, request count போன்ற numbers) சேகரிக்கும். CPU 90% cross ஆனால் alert அனுப்பும், Auto Scale trigger செய்யும். எல்லாவற்றையும் Dashboard-ல் visually பார்க்கலாம்.

Logs
Collect logs from EC2, ECS, Lambda, RDS. Search, filter, and analyze. "Find all 500 errors in the last hour." Log Insights for SQL-like queries on logs.
Metrics
Time-series data: CPU utilization, network in/out, disk I/O, custom app metrics. Stored for 15 months. Granularity: 1-second to 1-day.
Alarms
Alert when CPU > 80% for 5 minutes. Trigger SNS notification (SMS/email) or Auto Scaling action. "Page the on-call engineer at 3am."
Dashboards
Custom visual dashboards with charts, numbers, and logs. "War room" view during incidents. Share with your whole team in real-time.
# CloudWatch: Send custom metric from your app aws cloudwatch put-metric-data \ --namespace "MyApp/Orders" \ --metric-name "OrdersProcessed" \ --value 42 \ --unit Count # Create alarm: Alert if error rate > 1% for 5 minutes aws cloudwatch put-metric-alarm \ --alarm-name "HighErrorRate" \ --metric-name "5XXError" \ --namespace "AWS/ApplicationELB" \ --statistic Sum \ --period 300 \ --threshold 10 \ --comparison-operator GreaterThanThreshold \ --evaluation-periods 1 \ --alarm-actions "arn:aws:sns:ap-south-1:123:DevOpsAlert"

Deployment Pipeline CI/CD

# GitHub Actions — Full Deploy Pipeline name: Deploy to AWS EKS on: push: branches: [main] jobs: deploy: runs-on: ubuntu-latest steps: - uses: actions/checkout@v3 # 1. Build React frontend - name: Build React run: | cd frontend && npm ci && npm run build aws s3 sync build/ s3://myapp-frontend/ --delete aws cloudfront create-invalidation --distribution-id EXXXXX --paths "/*" # 2. Build & push Docker image - name: Build & Push API Image run: | aws ecr get-login-password | docker login --username AWS --password-stdin $ECR_URI docker build -t $ECR_URI/fastapi:${{ github.sha }} ./backend docker push $ECR_URI/fastapi:${{ github.sha }} # 3. Deploy to EKS - name: Deploy to EKS run: | aws eks update-kubeconfig --name prod-cluster --region ap-south-1 kubectl set image deployment/fastapi-deployment \ fastapi=$ECR_URI/fastapi:${{ github.sha }} kubectl rollout status deployment/fastapi-deployment
